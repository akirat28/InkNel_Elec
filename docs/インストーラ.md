# InkNel インストーラビルド手順

InkNel の macOS / Windows 配布用パッケージを作成する手順。次回リリース時はこの文書に沿って進める。

## 前提条件

### 共通
- Node.js (v20+)
- `npm install` 完了済み（postinstall で `electron-rebuild` が走るので、`better-sqlite3` が Electron 向けにビルドされている状態）

### macOS ビルド
- macOS 実機
- Apple Developer ID 証明書がキーチェーンに登録済み
  - 現在の identity: `akira Tanaka (F3P687R953)`
  - `package.json > build.mac.identity` で参照
- プロジェクトルートに `.env` を配置（git 管理外）

  ```env
  APPLE_ID=<Apple ID メールアドレス>
  APPLE_APP_SPECIFIC_PASSWORD=<App 用パスワード（appleid.apple.com で発行）>
  APPLE_TEAM_ID=F3P687R953
  ```

  - `APPLE_APP_PASSWORD` でも可（`scripts/notarize.cjs` が両方受理）
  - 未設定のままだと Notarization がスキップされ、SmartScreen/Gatekeeper 警告が出るビルドになる

### Windows ビルド
- 追加の設定不要（macOS 上からクロスビルド可能）
- 初回のみ `electron-builder` が Wine をダウンロードするため少し時間がかかる
- 署名は未設定（未署名ビルド → SmartScreen 警告が出る。個人配布なら問題なし）

## リリース手順

### 1. バージョンを更新

`package.json` の `version` を次のバージョンに書き換える（例: `0.1.5` → `0.1.6`）。

```json
{
  "version": "0.1.6"
}
```

### 2. テストで回帰確認

```sh
npm test
```

全 64 件（現状）がパスすることを確認。
`scripts/run-tests.cjs` が `better-sqlite3` を Node ↔ Electron 間でリビルドするため、終了後はそのまま次の手順へ進める。

### 3. 前回の release を削除

```sh
rm -rf release
```

古い成果物が混ざるのを防ぐため、毎回クリーンに始める。

### 4. macOS ビルド（署名 + Notarization）

```sh
set -a && source .env && set +a && npm run dist:mac:arm64
```

- `.env` を shell にロード → electron-builder が Developer ID で署名 → `afterSign` フックで `@electron/notarize` が Apple に提出
- 所要時間: 3〜5 分（Notarization 含む）
- 成果物:
  - `release/InkNel-<version>-arm64.dmg`
  - `release/InkNel-<version>-arm64-mac.zip`

ログ末尾に以下が出れば成功:
```
[notarize] Notarization 完了。
```

もし次のメッセージが出たら `.env` が読み込まれていない:
```
[notarize] APPLE_ID / APPLE_APP_PASSWORD(もしくは APPLE_APP_SPECIFIC_PASSWORD) 環境変数が未設定のため...
```

### 5. Windows ビルド（NSIS + ZIP）

```sh
npm run dist:win
```

- `@electron/rebuild` が `better-sqlite3` を win32-x64 向けにクロスリビルド
- Wine 経由で NSIS インストーラを生成
- 所要時間: 2〜3 分
- 成果物:
  - `release/InkNel Setup <version>.exe`（NSIS インストーラ）
  - `release/InkNel-<version>-win.zip`（ポータブル版）

### 6. web-site への配置

```sh
# 旧バージョン削除
rm web-site/downloads/InkNel-*.dmg web-site/downloads/InkNel-*.zip

# 新バージョン配置（配布は dmg + zip のみ）
cp release/InkNel-<version>-arm64.dmg web-site/downloads/
cp release/InkNel-<version>-win.zip   web-site/downloads/
```

### 7. index.html の更新

`web-site/index.html` 内の旧バージョン表記を置換する（計 3 箇所）:
- `Version X.Y.Z`（ヒーローバッジ）
- `downloads/InkNel-X.Y.Z-arm64.dmg`（Mac ダウンロードリンク）
- `downloads/InkNel-X.Y.Z-win.zip`（Windows ダウンロードリンク）

一括置換コマンド例:
```sh
# 旧バージョンから新バージョンへ置換
sed -i '' 's/0\.1\.5/0.1.6/g' web-site/index.html
```

### 8. ローカル dev 環境を復元

Windows ビルドで native module が win32-x64 向けに差し替わっているので、Mac arm64 向けに戻す:

```sh
npx electron-rebuild -f -w better-sqlite3
```

これで `npm run dev` がそのまま動く。

## 成果物まとめ（各リリースの配布サイズ目安）

| ファイル | サイズ | 用途 |
|---|---:|---|
| `InkNel-<ver>-arm64.dmg` | ~120 MB | Mac 配布（署名済み） |
| `InkNel-<ver>-arm64-mac.zip` | ~115 MB | 自動更新用（配布はしない） |
| `InkNel Setup <ver>.exe` | ~95 MB | Windows NSIS（現状配布していない） |
| `InkNel-<ver>-win.zip` | ~130 MB | Windows 配布（ポータブル） |

## トラブルシューティング

### Mac ビルドで Notarization に失敗する
- `.env` の `APPLE_ID` / `APPLE_APP_SPECIFIC_PASSWORD` / `APPLE_TEAM_ID` を再確認
- App 用パスワードはキーチェーンの「キーチェーンアクセス」でなく https://appleid.apple.com/account/manage で発行
- ネットワーク疎通が必要（Apple のサーバに提出するため）

### `Library not loaded: Electron Framework ... different Team IDs`
- `afterPack` の ad-hoc 署名と Developer ID 署名が競合している可能性
- `scripts/codesign-adhoc.cjs` は identity 設定時に自動スキップされる実装になっているので、通常は問題ないはず

### Windows ビルドで `better-sqlite3` が見つからない
- `@electron/rebuild` が x64 用 prebuilt の取得に失敗した可能性
- `rm -rf node_modules && npm install` してやり直す
- それでも失敗する場合は Windows 実機 or GitHub Actions `windows-latest` で `npm ci && npm run dist:win`

### Notarization が毎回成功するが、macOS で開けない
- `codesign --verify --deep --strict --verbose=2 /path/to/InkNel.app` で署名を検証
- `spctl --assess --verbose /path/to/InkNel.app` で Gatekeeper 評価
- 失敗する場合は `release/mac-arm64/InkNel.app` を消してから再ビルド

## 変更が生じやすい箇所

- **Apple ID / App 用パスワード**: 1 年ごとに再発行が必要な場合あり
- **Developer ID 証明書**: 5 年ごとに更新
- **`package.json > build.mac.identity`**: 証明書名が変わったら書き換え
- **Electron のバージョン更新**: `better-sqlite3` の prebuilt が揃っているか要確認（`NODE_MODULE_VERSION` が一致すること）

## 関連ファイル

- `package.json` — electron-builder 設定（`build` セクション）
- `scripts/codesign-adhoc.cjs` — Developer ID 証明書がない場合の ad-hoc 署名
- `scripts/notarize.cjs` — `afterSign` フック、Apple Notarization
- `scripts/patch-electron-name.cjs` — Electron.app → InkNel.app のリネーム（dev 時）
- `scripts/run-tests.cjs` — テスト前後の native module リビルドラッパー
- `build/entitlements.mac.plist` — Hardened Runtime 権限定義

# InkNel 詳細仕様（iOS / 他プラットフォーム共通リファレンス）

このドキュメントは、既存の **InkNel デスクトップ版（Electron + React）** の動作を余さず定義し、**React Native で iOS 版を新規構築する際の正**として参照するためのものです。クラウド同期のデータ形式を完全に共有することで、PC と iOS の両方で同じノートをシームレスに扱えるようにします。

## 1. プロダクト概要

### 1.1 目的

Markdown 形式のメモを書き留め、**複数デバイス間で自動同期**して参照・編集できるクロスプラットフォームのメモアプリ。

### 1.2 対応プラットフォーム

| プラットフォーム | ランタイム | 実装状況 |
|---|---|---|
| macOS / Windows | Electron 32.x + React 18 | リリース済み (v0.1.6) |
| iOS (iPhone / iPad) | React Native | **本仕様で新規構築** |
| Android | React Native | 将来対応（iOS と同一コードベース想定） |

### 1.3 データ共有の基本方針

- ユーザーのクラウド同期フォルダ（iCloud Drive / Dropbox / Google Drive）の `InkNel/` 配下を**共通ストレージ**として扱う
- **OAuth やクラウド API は使用しない**。OS の同期クライアントが持つローカル同期フォルダに直接 read/write する
- データ形式（`manifest.json` + `notes/<uuid>.md` + `images/` + `attachments/`）は全プラットフォームで共通
- 同期アルゴリズムは **`updatedAt` タイムスタンプベースの双方向同期**

## 2. アーキテクチャ

### 2.1 デスクトップ版（参考）

```
┌────────────────────────────────────────────────┐
│ Renderer (React + CodeMirror + markdown-it)    │
└──────────────┬─────────────────────────────────┘
               │ IPC（contextBridge）
┌──────────────┴─────────────────────────────────┐
│ Main Process (Electron)                        │
│   - SQLite (better-sqlite3)                    │
│   - ファイル I/O (userData/notes, images, ...) │
│   - クラウド同期エンジン                         │
└────────────────────────────────────────────────┘
```

### 2.2 iOS 版（想定構成）

```
┌────────────────────────────────────────────────┐
│ React Native (iOS)                             │
│  - UI: React Native + 独自 Markdown エディタ   │
│  - ストレージ: expo-sqlite / react-native-sqlite-storage │
│  - ファイル: expo-file-system                   │
│  - クラウド: expo-document-picker + iCloud      │
│             Document Browser (UIKit)           │
└────────────────────────────────────────────────┘
```

iOS では Electron のメインプロセス / レンダラ分離は存在しないが、**データアクセス層（DB + ファイル I/O + 同期）** を UI から独立した `services/` モジュールに分離することで、デスクトップ版と同等の責務分離を保つ。

## 3. データモデル

### 3.1 SQLite スキーマ

ユーザーデータディレクトリ配下の `inknel.db` に以下 3 テーブルを持つ。iOS でも同じスキーマを採用すること。

#### `notes`

| 列 | 型 | 制約 / 備考 |
|---|---|---|
| `id` | TEXT | PRIMARY KEY。UUIDv4 文字列 |
| `title` | TEXT | NOT NULL |
| `folder` | TEXT | NOT NULL DEFAULT ''。スラッシュ区切りパス（例 `work/ideas`） |
| `protected` | INTEGER | NOT NULL DEFAULT 0。編集時にパスワード要求（0/1） |
| `secret` | INTEGER | NOT NULL DEFAULT 0。閲覧時にもパスワード要求（0/1） |
| `tags` | TEXT | NOT NULL DEFAULT '[]'。JSON 配列文字列 |
| `created_at` | INTEGER | NOT NULL。epoch ミリ秒 |
| `updated_at` | INTEGER | NOT NULL。epoch ミリ秒。同期比較の基準 |

**インデックス**: `idx_notes_folder(folder)`, `idx_notes_updated(updated_at)`

#### `folders`

空フォルダ（ノートを持たないフォルダ）を記録する。ノートを持つフォルダは `notes.folder` から自動派生。

| 列 | 型 | 制約 |
|---|---|---|
| `path` | TEXT | PRIMARY KEY。スラッシュ区切り |
| `created_at` | INTEGER | NOT NULL |

#### `settings`

| 列 | 型 | 制約 |
|---|---|---|
| `key` | TEXT | PRIMARY KEY |
| `value` | TEXT | NOT NULL |

### 3.2 ファイル本体 (.md)

SQLite とは別に、各ノートの本文 Markdown を `notes/<id>.md` としてファイルシステムに保存する。

- ファイル名は UUID（拡張子 `.md`）
- エンコーディング UTF-8、改行 LF
- ファイルシステム上は**階層化しない**（フラット配置）。階層は `notes.folder` で論理的に管理

### 3.3 メディアファイル

#### `images/`

Markdown 中に貼り付けた画像を SHA-256 でコンテンツアドレス化して保存。

- ファイル名: `<sha256>.<ext>`（例 `a1b2...<全64桁>.png`）
- 許可拡張子: `png / jpg / jpeg / gif / webp / svg / avif / bmp`
- 上記以外は `.bin` にフォールバック
- サイズ上限: **25 MB / 枚**（デスクトップ版）
- 同一バイナリは自動 dedupe（同じ SHA-256 → 同じファイル名）

#### `attachments/`

ノートにリンクする添付ファイル。

- 命名規則は `images/` と同じ
- 許可拡張子: `pdf / zip / lzh / lha / 7z`
- 以外は `.bin`
- サイズ上限: **100 MB / ファイル**

### 3.4 Markdown 内のメディア参照形式

ノート本文中では次の相対パスで参照する。プラットフォーム間で互換。

- 画像: `![代替テキスト](images/<sha256>.<ext>)`
- 添付: `[ラベル](attachments/<sha256>.<ext>)`

表示時は各プラットフォームのカスタムプロトコル等で解決（デスクトップ版は `inknel-image://` を使用）。iOS では RN 側のローカルファイル URI（`file://`）に変換して表示する。

### 3.5 タグ

本文中の `#word` とノートメタの `tags` 配列の 2 系統。検索 / 分類で併用。

- 本文タグの正規表現: `/(?:^|\s)#([\p{L}\p{N}_-]+)/gu`
- 見出し行 (`# ` 〜 `###### `) とコードブロック内は除外
- メタの `tags` は TagBar で手動編集（JSON 配列で保存）

## 4. ローカルストレージ

### 4.1 ユーザーデータディレクトリ

| プラットフォーム | パス |
|---|---|
| macOS | `~/Library/Application Support/InkNel/` |
| Windows | `%APPDATA%\InkNel\` |
| iOS | `{FileSystem.documentDirectory}/` (`expo-file-system`) |
| Android | `{FileSystem.documentDirectory}/` |

### 4.2 ディレクトリ構成

```
<userData>/
├── inknel.db             # SQLite: notes / folders / settings
├── inknel.db-wal         # SQLite WAL（自動）
├── inknel.db-shm         # SQLite shared memory（自動）
├── notes/
│   └── <uuid>.md
├── images/
│   └── <sha256>.<ext>
└── attachments/
    └── <sha256>.<ext>
```

WAL モード有効 (`PRAGMA journal_mode = WAL`)、外部キー有効 (`PRAGMA foreign_keys = ON`)。

### 4.3 未参照メディアの GC

編集モード → プレビューモード切替時にのみ発生。

1. 編集開始時に本文中の画像 / 添付ファイル名を `sessionImagesRef / sessionAttachmentsRef` に記録
2. プレビューに切り替える際に、セッション中に存在したが現在は無い → GC 候補
3. GC 候補のうち、**どのノートからも参照されていない**ものだけ物理削除

iOS でも同じロジックを実装する。

## 5. クラウド同期プロトコル（PC / iOS 共通）

### 5.1 対応プロバイダ

各プロバイダの**ローカル同期フォルダ**を経由する。OAuth やクラウド API は使わない。

| プロバイダ | macOS 検出 | Windows 検出 | iOS 検出 |
|---|---|---|---|
| iCloud Drive | `~/Library/Mobile Documents/com~apple~CloudDocs/` | `%USERPROFILE%\iCloudDrive\` | `UIDocumentBrowserViewController` でユーザー選択、または `NSUbiquityContainer` |
| Dropbox | `~/.dropbox/info.json → path` | 同等 | Dropbox iOS SDK またはファイル選択 |
| Google Drive | `~/Library/CloudStorage/GoogleDrive-<email>/<DriveName>/` | 同等 | Google Drive iOS SDK または PWA |

**複数同時利用不可**。設定画面で 1 つだけ選択する。

### 5.2 同期ルート

選択されたプロバイダのローカルフォルダ直下に `InkNel/` を作成し、以下を配置。

```
<cloud-root>/InkNel/
├── manifest.json
├── notes/
│   └── <uuid>.md
├── images/
│   └── <sha256>.<ext>
└── attachments/
    └── <sha256>.<ext>
```

### 5.3 manifest.json スキーマ

```json
{
  "version": 1,
  "lastSync": 1712851200000,
  "notes": {
    "<uuid>": {
      "title": "タイトル",
      "folder": "work/ideas",
      "protected": false,
      "secret": false,
      "tags": ["tag1", "tag2"],
      "createdAt": 1712800000000,
      "updatedAt": 1712850000000
    }
  }
}
```

- `version` は `1` 固定（将来の構造変更に備えた識別子）
- 不正な JSON / 欠損時は**新規扱い**で初期化（既存クラウドデータは保護のため上書きせず、次回 push で再構築）
- iOS 版も同じフォーマットで読み書きすること

### 5.4 同期アルゴリズム

#### `updatedAt` ベース双方向同期

| PC の `updatedAt` | クラウドの `updatedAt` | アクション |
|---|---|---|
| 同じ | 同じ | unchanged |
| **PC のが新しい** | 古い | **push**（PC → クラウド） |
| 古い | **クラウドのが新しい** | **pull**（クラウド → PC、DB 更新） |
| PC にしかない | — | push |
| — | クラウドにしかない | pull（DB 新規挿入） |

#### フル同期 (`runSync`)

起動時または手動実行で呼び出される。

1. `manifest.json` をロード（無ければ空で開始）
2. ローカル DB から全ノートを取得
3. 和集合の全 ID を走査 → 上記ルールで push / pull / unchanged を決定
4. メディアディレクトリ (`images/`, `attachments/`) をファイル名ベースで差分同期（SHA-256 同名 = 同一内容 → skip）
5. `manifest.lastSync = Date.now()` に更新して保存

#### ライトスルー（即時 push）

ノートの mutation 直後に対応する単一ノートだけをクラウドに書き戻す。

- `notes:create` → `pushSingleNote`
- `notes:update-meta` → `pushSingleNote`
- `notes:update-body` → `pushSingleNote`
- `notes:set-protected` / `notes:set-secret` → `pushSingleNote`
- `notes:delete` → `removeSingleNote`
- `images:save` / `attachments:save` → `pushSingleMedia`

これにより、アプリを閉じるまで同期を待たずに他デバイスから見える。

#### 単一ノートチェック (`checkAndSyncSingleNote`)

ノート選択時にバックグラウンドで呼ばれる。

- PC だけにある → push
- クラウドが新しい → pull（呼び出し元で本文再読込が必要、戻り値 `'pulled'`）
- PC が新しい → push（戻り値 `'pushed'`）
- 同じ → `'same'`
- プロバイダ無効/検出不可 → `'skip'`

### 5.5 削除の扱い

- manifest から削除された ID は「他デバイスで削除された」と見なし、フル同期時にローカルからも削除する
- 削除されたノート ID は manifest 側に tombstone として残さない（容量とシンプルさ優先）
- 完全同期は manifest が**唯一の真実**として働く

### 5.6 競合解決

- 同じノートを 2 デバイスで同時編集した場合、**後に updatedAt が新しい方で上書き**（Last-Write-Wins）
- 手動マージ UI は提供しない
- 必要に応じて将来、バージョン履歴を追加することで対応可能

## 6. 機能仕様

### 6.1 ノート管理

| 操作 | トリガー | 挙動 |
|---|---|---|
| 新規作成 | サイドバー 📄+ ボタン / ⌘N / File メニュー | 現在選択中のノートと同じフォルダに無題ノートを作成、タブに追加、edit モードで開く |
| 選択 | サイドバーのファイル行クリック / タブクリック | 該当ノートを読み込み、preview モードで表示 |
| タイトル変更 | NoteHeader の入力欄 / 右クリック → 名称変更 | ファイル名入力に `/` を含めるとフォルダ移動 |
| 本文編集 | Editor（CodeMirror 6） | 300ms デバウンスで自動保存、ライトスルーで即時クラウド push |
| 保護 ON | ケバブ → 🔒 保護 | パスワード不要、即時 ON |
| 保護 OFF | ケバブ → 🔓 保護解除 | パスワードダイアログ → 正解で OFF |
| シークレット ON | ケバブ → 👁 シークレット | パスワード不要、即時 ON |
| シークレット OFF | ケバブ → 👁 シークレット解除 | パスワード要求 |
| 削除 | ケバブ → 🗑 削除 | 保護中は不可（エラー通知）。タブからも除去 |

### 6.2 フォルダ管理

- ツリー表示、チェブロンで展開/折りたたみ
- **展開状態は永続化**（`sidebar.expanded` キーで保存、300ms デバウンス）
- ドラッグ&ドロップでノート / フォルダを他階層へ移動
  - 自身・自身の子孫へのドロップは禁止（循環防止）
- 新規フォルダボタンで空フォルダを作成
- 「ディレクトリごと削除」で配下のノート＋サブフォルダ一括削除（保護ノートが 1 件でも含まれると例外）

### 6.3 検索

- 検索対象: タイトル + 本文（ケースインセンシティブ）
- ヒット順: タイトル一致 → 本文一致
- 履歴: `reset` モード（セッション限定）/ `persist` モード（DB 保存）
- 履歴上限: 100 / 1000 件から選択

### 6.4 タグ

- 本文中の `#word` とメタの `tags` 配列の両方を集計
- タグパネルでタグ → ヒットノート一覧を表示

### 6.5 インポート

- **ファイル読み込み**: `.md` / `.markdown` の複数選択 → `読み込みファイル/` フォルダに追加
- **ディレクトリ読み込み**: 再帰走査し `.md` / `.markdown` を取り込み、元のサブフォルダ階層を再現 → `読み込みファイル/<ディレクトリ名>/<サブ>/` の形
- 進捗バーをフッターに表示

### 6.6 テンプレート

- 設定で指定した「テンプレートフォルダ」（既定: `template`）配下のノートをテンプレートとして扱う
- EditorToolbar から一覧をポップアップ、選択して本文に挿入
- テンプレートフォルダは**最上位階層のみ対応**（`template/` は OK、`foo/template/` は NG）

### 6.7 タブ UI

- メイン領域上部にタブバーを表示
- 1 ノート = 1 タブ、**同じノートは既に開いていればそのタブにフォーカス**（重複なし）
- タブ個別に × で閉じる / ミドルクリックで閉じる
- **タブごとに view モード（edit/preview）が独立**
- **タブごとに保護解錠状態が独立**（タブを閉じると解錠解除）
- **起動時にタブ状態を復元**（`ui.openTabs` / `ui.activeTab` を永続化）
- 並び替えは未対応（将来拡張）

### 6.8 印刷

- CmdOrCtrl+P で現在のノートを印刷
- Preview の HTML をそのまま印刷ダイアログに

### 6.9 空状態

`openTabIds.length === 0` のとき、メイン領域中央に羽ペンアイコン + "InkNel" + "Markdown で思考を整理する" のタグラインを表示。

| 変数 | dark | light |

| `--fg` | `#d4d4d4` | `#1e1e1e` |

### 7.4 フォント

| 設定キー | 用途 | 選択肢 |
|---|---|---|
| `fontFamily` | ノート本文 | system / sans / serif / mono |
| `fontSize` | ノート本文 | 12-20 |
| `sidebarFontFamily` | サイドバー | 同上 |
| `sidebarFontSize` | サイドバー | 同上 |

iOS の場合、system は `-apple-system`、sans は `"Hiragino Sans"`、serif は `"Hiragino Mincho ProN"`、mono は `"SF Mono"` などに対応付ける。

## 8. マークダウン処理

### 8.1 使用エンジン

- デスクトップ: `markdown-it` (JS)
- iOS: RN 用に `react-native-markdown-display` / `react-native-marked` いずれか。ただし **拡張仕様（下記）を揃える**必要があるため、同じ markdown-it を WebView 内で使うのが最も互換性が高い

### 8.2 必須拡張

| 機能 | 備考 |
|---|---|
| 見出し id 自動付与 | `slugifyForAnchor(text)` で日本語対応スラッグ。重複時 `-2`, `-3` サフィックス |
| 見出しアンカーリンク (`[#見出し](#見出し)`) | 見出し内部スクロール |
| タスクリスト (`- [ ]` / `- [x]`) | チェックボックス描画、クリックでトグル → 本文書換 |
| テーブル (GFM) | 標準 |
| ハードラップ (`breaks: true`) | 改行を `<br>` に |
| コードブロックのシンタックスハイライト | `highlight.js` |
| コードブロックのコピーボタン / 言語ラベル / 行番号 | 設定で ON/OFF |
| 画像カスタムパス解決 | `images/<sha256>.<ext>` → プラットフォーム固有 URI |
| 外部リンク `target="_blank"` | クリック時は `shell.openExternal` / `Linking.openURL` |
| `html: false` | 生 HTML は許可しない（XSS 対策） |

### 8.3 slugifyForAnchor 仕様

```ts
// 日本語を含む見出しテキストから URL 用スラッグを生成
function slugifyForAnchor(text: string): string {
  return text
    .toLowerCase()
    .replace(/[^\p{L}\p{N}\s-]/gu, '') // 英数字・全言語の文字・空白・ハイフン以外を除去
    .trim()
    .replace(/\s+/g, '-'); // 空白を - に
}
```

重複したスラッグは `slug`, `slug-2`, `slug-3` のように採番。

### 8.4 編集モード

- CodeMirror 6 + `@codemirror/lang-markdown`
- iOS では RN のネイティブ `TextInput` をベースに、シンタックスハイライトを自前実装するか、WebView で CodeMirror を呼び出す構成を推奨

## 9. セキュリティ / 保護機構

### 9.1 パスワード

- **4 桁の数字**のみ（正規表現 `/^\d{4}$/`）
- デフォルト: `1234`
- 設定画面で変更可能（旧パスワードを入力 → 新パスワードで上書き）
- DB の `settings` テーブルに平文で保存（設計判断: ローカル限定、紛失時のリカバリ優先）

### 9.2 保護 (protected)

- 閲覧（preview）は自由
- 編集モードに切り替える際にパスワード要求
- タブを閉じる / 別プロセス起動 で再度ロック
- 同一タブ内では解錠状態を維持

### 9.3 シークレット (secret)

- **閲覧時点でパスワード要求**（サイドバーで選択した瞬間）
- 解錠後はセッション中維持（`unlockedSecretIds: Set`）
- 保護 + シークレットを同時に ON にした場合、1 回のパスワードで両方解錠

### 9.4 マトリクス

| 保護 | シークレット | 選択時 | edit 切替時 |
|---|---|---|---|
| OFF | OFF | そのまま表示 | そのまま編集 |
| ON | OFF | そのまま表示 | **パスワード要求** |
| OFF | ON | **パスワード要求** | パスワード要求なし |
| ON | ON | パスワード要求（両方解錠） | 解錠済み |

### 9.5 Electron セキュリティ設定

- `contextIsolation: true`
- `nodeIntegration: false`
- `sandbox: false`（preload で一部 Node API 必要のため）
- カスタムプロトコル `inknel-image://` でメディアファイル配信

iOS 版では WebView を使う場合、同等の隔離を `WKWebView` の設定で担保する。

## 12. エラーハンドリング方針

### 12.1 原則

- ユーザー入力に関するエラーは `window.alert`（デスクトップ）or `Alert.alert`（iOS）で通知
- 同期エラーは**フッターに表示**してアプリ動作は継続（再試行でリカバリ可能）
- ネットワーク系は明示的に無視（next sync で追いつく）
- ログレベル: `console.error` / `console.warn` / `console.log`

### 12.2 代表例

| ケース | 挙動 |
|---|---|
| 保護ノートの削除試行 | アラート「保護されているノートは削除できません」 |
| パスワード不正 | ダイアログ内エラー表示、retry |
| 画像サイズ超過 (>25MB) | アラート、保存拒否 |
| 添付サイズ超過 (>100MB) | アラート、保存拒否 |
| ファイル名に `../` | フォルダパス正規化で除去（静かに） |
| クラウドフォルダ検出失敗 | プロバイダ無効扱いに自動フォールバック |
| manifest.json 破損 | 新規扱いで初期化 |

## 13. iOS 実装時の考慮事項

### 13.1 ライブラリ候補

| 用途 | 候補 |
|---|---|
| ランタイム | Expo (React Native) |
| SQLite | `expo-sqlite` |
| ファイル I/O | `expo-file-system` |
| 暗号化ハッシュ | `expo-crypto` (SHA-256) |
| ドキュメントピッカー | `expo-document-picker` |
| iCloud Drive | `expo-file-system` の `FileSystem.StorageAccessFramework` + `UIDocumentBrowserViewController` (カスタムネイティブモジュール) |
| Markdown レンダラ | WebView + markdown-it（互換性優先）<br>または `react-native-marked` |
| Markdown エディタ | WebView + CodeMirror 6<br>またはネイティブ TextInput + 独自構文強調 |
| 状態管理 | React Context + useReducer、または Zustand |
| ナビゲーション | React Navigation (bottom tabs + stack) |

### 13.2 iCloud 連携の注意点

iOS では macOS のように `~/Library/Mobile Documents/com~apple~CloudDocs/` へ直接アクセスできない。代わりに:

1. **Ubiquity Container** を設定した場合: `FileManager.default.url(forUbiquityContainerIdentifier:)` で iCloud Drive 配下の App 専用領域にアクセス可能（他のアプリからは直接見えない）
2. **Files アプリ経由**: `UIDocumentBrowserViewController` でユーザーに `InkNel/` フォルダを選択させ、ブックマークを保持する方式

同期フォーマットは共通のため、どちらの方式でも manifest.json / notes / images / attachments のレイアウトを守ればデスクトップ版と相互運用可能。

### 13.3 バックグラウンド同期

- iOS では長時間のバックグラウンド実行は制限される
- 戦略: **フォアグラウンド復帰時 / アプリ起動時 / ノート選択時**に同期
- デスクトップ版と同じ `runSync` / `checkAndSyncSingleNote` / ライトスルー 3 層構成

### 13.4 Markdown エディタ実装

iOS のフォームサブクラスとの相性の都合で、以下 2 案を比較する:

| 案 | メリット | デメリット |
|---|---|---|
| WebView + CodeMirror 6 | デスクトップ版と挙動が揃う、拡張が楽 | パフォーマンス若干落ちる、WebView のメモリ使用 |
| ネイティブ TextInput + 独自ハイライト | 軽量・ネイティブ操作感 | 実装コスト、エッジケース多い |

**推奨: WebView + CodeMirror**（コード資産を最大限再利用し、プレビューも同じ markdown-it を使える）

### 13.5 UUID 生成

- デスクトップ: `crypto.randomUUID()` (Node 20+)
- iOS: `expo-crypto` の `Crypto.randomUUID()` または `uuid` パッケージ
- 形式は RFC 4122 v4 で揃える

### 13.6 ファイル名検証

クラウドから来る信頼されていないファイル名は以下パターンで厳格検証:

- 画像: `/^[a-f0-9]{64}\.[a-z0-9]{2,5}$/`
- 添付: 同上

ディレクトリトラバーサル（`../`, バックスラッシュ）は絶対に許可しない。

### 13.7 フォルダパスの正規化

```ts
function normalizeFolderPath(input: string): string {
  if (typeof input !== 'string') return '';
  return input
    .split('/')
    .map((s) => s.trim())
    .filter((s) => s.length > 0 && s !== '.' && s !== '..' && !s.includes('\\'))
    .join('/');
}
```

iOS でもこの実装を踏襲すること。

## 14. 用語集

| 用語 | 意味 |
|---|---|
| **ノート (Note)** | 1 つの Markdown メモ。UUID で識別 |
| **フォルダ (Folder)** | ノートを整理するスラッシュ区切りの仮想階層 |
| **保護 (Protected)** | 編集時にパスワードを要求するフラグ |
| **シークレット (Secret)** | 閲覧時にもパスワードを要求するフラグ |
| **プロバイダ (Provider)** | クラウド同期先（iCloud / Dropbox / Google Drive / none） |
| **ライトスルー (Write-through)** | mutation 直後にクラウドへも即時書き込む同期方式 |
| **フル同期 (Full Sync)** | 全ノートを一括で双方向同期する処理 |
| **manifest.json** | クラウド側のメタデータインデックス |
| **SHA-256 命名** | メディアファイルをバイト内容のハッシュでファイル名にする方式（自動 dedupe） |
| **タブ (Tab)** | 開いているノートを表す UI 要素 |
| **アクティブタブ** | 現在フォーカスされているタブ |

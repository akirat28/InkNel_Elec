# InkNel 画面構成リファレンス

このドキュメントは InkNel の UI 構成を整理したもので、**「○○エリアの□□に××を追加して」のような指示を出すときの参照** として使用する。各エリアには **正式名称**（このドキュメントで使う呼び方）を定めている。

最新化基準: v0.1.15

---

## 1. 全体レイアウト

ウィンドウ全体は **横方向の4カラム**（最右の AI チャットは閉じている時は非表示）+ メインに **縦方向の積層**。

```
┌──────┬──────────────┬─────────────────────────────────┬──────────┐
│      │              │ TabBar                          │          │
│ ア   │              ├─────────────────────────────────┤          │
│ ク   │              │ NoteHeader                      │   AI     │
│ テ   │   サイドバー │  ┌────────────────────────────┐ │ チャット │
│ ィ   │ (Sidebar)    │  │ EditorToolbar              │ │ ペイン   │
│ ビ   │              │  ├────────────────────────────┤ │ (任意)   │
│ テ   │              │  │ TagBar                     │ │          │
│ ィ   │              │  ├────────────────────────────┤ │ (右端    │
│ バ   │              │  │                            │ │  リサイ  │
│ │    │              │  │   Editor / Preview         │ │  ザー)   │
│      │              │  │                            │ │          │
│      │              │  └────────────────────────────┘ │          │
└──────┴──────────────┴─────────────────────────────────┴──────────┘
  48px    可変幅 (160-480px)            残り全幅              280-560px
                                                          (折りたたみ可)
```

| ID | 名称 | 幅 | 役割 |
|---|---|---|---|
| A | **アクティビティバー** (`ActivityBar`) | 48px 固定 | ビューモード切替（ファイル / 検索 / タグ / 保存先と同期 / 設定） |
| B | **サイドバー** (`Sidebar`) | 可変 (160-480px) | ファイル一覧ツリー / 検索パネル / タグパネル / 保存先同期パネル |
| C | **メイン領域** (`Main Area`) | 残り全幅 | タブバー + 選択中ノートの表示・編集 |
| D | **AI チャットペイン** (`AiChatPanel`) | 可変 (折りたたみ可) | LLM とのチャット（任意で開閉） |

サイドバー右端のドラッグハンドルで幅を調整可能。アクティビティバーの「ファイル」「検索」「タグ」「保存先と同期」アイコンを再クリックでサイドバーを折りたたみ。

## 2. アクティビティバー (`ActivityBar`)

画面の最左端、48px 幅の縦アイコン列。VS Code 風。

```
┌──────┐
│      │
│  📄  │ ← ファイル
│      │
│  🔍  │ ← 検索
│      │
│  🏷  │ ← タグ
│      │
├──────┤  (上下グループ区切り)
│      │
│  💾  │ ← 保存先と同期 (HDD)
│      │
│  ⚙   │ ← 設定（最下部）
│      │
└──────┘
```

| アイコン | 名称 | クリック動作 |
|---|---|---|
| 📄 | **ファイルアイコン** | サイドバーをファイルモードに / 既にファイルモードなら折りたたみトグル |
| 🔍 | **検索アイコン** | サイドバーを検索モードに / 既に検索モードなら折りたたみトグル |
| 🏷 | **タグアイコン** | サイドバーをタグモードに / 既にタグモードなら折りたたみトグル |
| 💾 | **HDD アイコン** | サイドバーを「保存先と同期」モードに / 既にそのモードなら折りたたみトグル |
| ⚙ | **設定アイコン**（最下部固定） | 設定モーダルを開く（サイドバーモードは変更しない） |

**ファイル**: `src/components/ActivityBar.tsx`
**CSS クラス**: `.activity / .activity__btn / .activity__btn.is-active`

## 3. サイドバー (`Sidebar`)

中央のカラム。アクティビティバーの選択により4モードを切り替え。

### 3.1 共通要素

```
┌──────────────┐
│ ファイル  📄+📁+│ ← サイドバーヘッダー (sidebar__header)
├──────────────┤
│              │
│   本体       │
│              │
│              │
└──────────────┘
              ↑右端: ドラッグハンドル (sidebar__resizer)
```

| 要素 | 名称 | 備考 |
|---|---|---|
| ヘッダーのタイトル | **サイドバータイトル** | "ファイル" / "検索" / "タグ" / "保存先と同期" |
| 右端のリサイズハンドル | **サイドバーリサイザー** | 左右ドラッグで幅変更 |

### 3.2 ファイルモード (mode = `'files'`)

```
┌──────────────┐
│ ファイル 📄+📁+│ ← 「+ ボタン群」: 📄+ 新規ノート / 📁+ 新規フォルダ
├──────────────┤
│ ▼ 📁 階層1     │ ← フォルダ行 (tree__folder)
│   📄 ノートA  ⋮│ ← ファイル行 (tree__file)、⋮はホバー時のみ
│   ▼ 📁 階層2  ⋮│
│     📄 ノートB🔒⋮│ ← 🔒は保護中 (tree__lock-indicator)
│     📄 ノートC👁⋮│ ← 👁 はシークレット (tree__secret-indicator)
│ 📄 はじめに   ⋮│
└──────────────┘
```

| 要素 | 名称 | 場所 |
|---|---|---|
| 📄+ ボタン | **新規ノートボタン** | sidebar__header 右側 |
| 📁+ ボタン | **新規フォルダボタン** | sidebar__header 右側 |
| ▼/▶ | **チェブロン** (tree__chevron) | フォルダ行先頭、展開状態 |
| 📁 アイコン | **フォルダアイコン** (tree__icon) | フォルダ行 |
| 📄 アイコン | **ファイルアイコン** (tree__icon) | ファイル行 |
| ⋮ ボタン | **ケバブボタン** (tree__menu-btn) | 行右端、ホバー時のみ表示 |
| 🔒 アイコン | **錠前インジケータ** (tree__lock-indicator) | 保護中ファイルの右端 |
| 👁 アイコン | **シークレットインジケータ** (tree__secret-indicator) | シークレット中ファイルの右端 |

**ケバブメニュー（ファイル）の項目** — OS ネイティブメニュー、ウィンドウ外まで展開可:
1. 名称変更
2. 保護 / 保護解除
3. シークレットにする / シークレット解除
4. （区切り線）
5. 削除（保護中はグレーアウト）

**ケバブメニュー（フォルダ）の項目**:
1. ノートの作成（このフォルダ配下に「無題」を作成）
2. 名称変更（配下を一括更新）
3. （区切り線）
4. ディレクトリごと削除

**右クリック動作**: ファイル行 / フォルダ行を右クリックすると、ケバブメニューと同じメニューがカーソル位置に開く。

**操作**:
- ファイル → フォルダへ **ドラッグ&ドロップで移動**
- フォルダ → 別フォルダへ **ドラッグ&ドロップで移動**（配下のノート / サブフォルダも追従）
- フォルダの ▼ クリックで展開/折りたたみ
- フォルダの展開状態は **永続化**（次回起動時に復元）

### 3.3 検索モード (mode = `'search'`)

```
┌──────────────┐
│ 検索          │
├──────────────┤
│ [入力欄][検索]│ ← 検索フォーム (search-panel__form)
├──────────────┤
│ 3 件         │
│ ┌──────────┐ │
│ │ノ[ート]A 🔒│ │ ← 検索結果アイテム（[]部分は <mark> でハイライト）
│ │work/proj │ │
│ └──────────┘ │
│ ┌──────────┐ │
│ │別ノ[ート] │ │
│ └──────────┘ │
└──────────────┘
```

| 要素 | 名称 | 備考 |
|---|---|---|
| 入力欄 | **検索入力欄** | ↑↓ で過去の履歴をたどれる |
| 検索ボタン | **検索実行ボタン** | Enter でも実行可 |
| 結果リスト | **検索結果リスト** | タイトル一致 → 本文一致の順 |
| 黄色ハイライト | **検索ヒット強調** (`.search-hit`) | キーワードに一致した部分を黄色背景 + 太字で強調 |

### 3.4 タグモード (mode = `'tags'`)

```
┌──────────────┐
│ タグ          │
├──────────────┤
│ ▶ 🏷 #important  3 │ ← タグ行 (tags-list__row) 折りたたみ状態
│ ▼ 🏷 #todo       2 │ ← 展開状態
│     📄 ノートA       │ ← 配下のノート行 (tree__file)
│     📄 ノートB       │
│ ▶ 🏷 #idea       5 │
└──────────────┘
```

| 要素 | 名称 | 備考 |
|---|---|---|
| タグ行 | **タグ行** (tags-list__row) | クリックでアコーディオン展開/折りたたみ |
| 件数バッジ | **タグ件数バッジ** (tags-list__count) | そのタグを含むノート数 |
| 配下のノート行 | **タグ配下ノート行** (tree__file) | クリックでノートを開く |

**タグの定義**:
- **TagBar で明示的に設定したタグのみ**が対象
- 本文中の `#word` 自動収集は廃止（v0.1.10〜）

### 3.5 保存先と同期モード (mode = `'sync'`)

```
┌──────────────────┐
│ 保存先と同期      │
├──────────────────┤
│ 保存先フォルダ:   │
│ /Users/.../iCloud │
│ /InkNel           │
│ DB ノート数: 42   │
│ ディスク .md: 41  │
│ 差分:             │
│  書き出し候補 1   │
│  取り込み候補 0   │
│                  │
│ [同期][再スキャン]│
└──────────────────┘
```

| 要素 | 名称 | 役割 |
|---|---|---|
| 保存先パス | `sync-panel__path` | 現在の `<storage-root>` |
| 統計 | DB ノート数 / ディスクファイル数 / 差分件数 |
| 同期ボタン | DB ↔ disk の差分を双方向反映 |
| 再スキャンボタン | 統計を再取得 |

**ファイル**:
- `src/components/Sidebar.tsx` (本体・ツリー描画)
- `src/components/SearchPanel.tsx` (検索モード)
- `src/components/TagsPanel.tsx` (タグモード)
- `src/components/StorageSyncPanel.tsx` (保存先と同期モード)

## 4. メイン領域 (`Main Area`)

選択中のノートを表示・編集する右側の領域。**縦に5つのブロック**を積層。

```
┌────────────────────────────────────────┐
│ TabBar                                 │ ← 4.1 (新規追加 v0.1.7〜)
├────────────────────────────────────────┤
│ NoteHeader                             │ ← 4.2
├────────────────────────────────────────┤
│ EditorToolbar (編集モード時のみ)      │ ← 4.3
├────────────────────────────────────────┤
│ TagBar (編集モード時のみ)             │ ← 4.4
├────────────────────────────────────────┤
│                                        │
│   Editor (編集)  または  Preview      │ ← 4.5
│                                        │
└────────────────────────────────────────┘
```

タブが 1 つも開かれていない時は、TabBar 以外の代わりに **空状態 (`empty-state`)** が表示される（中央に羽ペン風 InkNel ロゴ + キャッチコピー「Markdown で思考を整理する」）。

### 4.1 タブバー (`TabBar`)

```
┌────────────────────────────────────────┐
│ « │ ノートA × │ ノートB ×│ ノートC × │ » │
└────────────────────────────────────────┘
   ↑               ↑↑              ↑       ↑
  左へ        アクティブタブ      閉じる    右へ
  スクロール  (青下線)            ボタン   スクロール
  (オーバーフロー時のみ)
```

| 要素 | 名称 | 役割 |
|---|---|---|
| タブ | **タブ** (tab) | 1 ノート = 1 タブ。タイトル + × 閉じるボタン |
| アクティブタブ | **`tab--active`** | 現在選択中。下に `--accent` カラーの 2px 線 |
| × ボタン | **タブの閉じるボタン** (tab__close) | 個別に閉じる |
| « / » | **タブスクロールナビ** (tab-bar__nav) | タブが横幅を超えた時のみ表示。SCROLL_STEP=200px ずつスクロール |

**動作**:
- ノート選択 → タブ追加（既に開いていればフォーカス）
- **タブのドラッグ&ドロップ** で並び替え（同一タブバー内）
- タブの **右クリック** で OS ネイティブのコンテキストメニュー:
  1. このタブを閉じる
  2. 他を閉じる
  3. すべてのタブを閉じる
  4. 右のタブをすべて閉じる
- ミドルクリックで個別に閉じる
- **アクティブタブが画面外**にある時は自動的に可視位置までスクロール
- 開いているタブ ID 一覧は `ui.openTabs` 設定に永続化、再起動時に復元
- アクティブタブも `ui.activeTab` で永続化

**ファイル**: `src/components/TabBar.tsx`
**CSS**: `.tab-bar / .tab-bar__list / .tab / .tab--active / .tab--dragging / .tab--drop-before / .tab--drop-after / .tab__close / .tab-bar__nav`

### 4.2 ノートヘッダー (`NoteHeader`)

```
┌────────────────────────────────────────┐
│ [ファイル名 (例: 階層1/テスト1)] [✎][⋮]│
└────────────────────────────────────────┘
```

| 要素 | 名称 | 役割 |
|---|---|---|
| 入力欄 | **ファイル名入力欄** (note-header__name) | スラッシュ区切りパス入力。例: `階層1/テスト1` |
| ✎/👁 ボタン | **編集 / プレビュートグル** (view-toggle__btn) | 表示モードを切替（次の状態のアイコンを表示） |
| ⋮ ボタン | **ノートヘッダーのケバブボタン** | OS ネイティブメニューを開く |

**ケバブメニューの項目**:
1. PDF で出力（保存ダイアログ → printToPDF）
2. Markdown で出力（保存ダイアログ → 本文を `.md` で保存）
3. （区切り線）
4. 印刷（OS の印刷ダイアログ）

ファイル名はスラッシュで分割され、最後のセグメントが title、それ以前が folder として保存される。

**ファイル**: `src/components/NoteHeader.tsx`

### 4.3 編集ツールバー (`EditorToolbar`) — 通称「タグボタンリスト」

編集ビュー時のみ NoteHeader 直下に表示される、マークダウン挿入ボタンの一覧。
**4つのグループ** を区切り線で分けて配置。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ H1 H2 H3 H4 H5 H6 │ B I S <> │ • 1. ☐ ❝ │ 🔗 {} ― ▦ ⇄ 😊 📅 📋          │
└─────────────────────────────────────────────────────────────────────────────┘
   見出しグループ    強調グループ  リストグループ  挿入グループ
```

| グループ名 | 含まれるボタン | 動作 |
|---|---|---|
| **見出しグループ** | H1〜H6 | 行頭に `#` 〜 `######` を追加 |
| **強調グループ** | **B** (太字) / *I* (斜体) / ~~S~~ (取消) / `<>` (コード) | 選択範囲を記号で囲む |
| **リストグループ** | • (箇条書き) / 1. (番号) / ☐ (タスク) / ❝ (引用) | 行頭に各 prefix を追加 |
| **挿入グループ** | 🔗 (リンク) / `{}` (コードブロック) / ― (区切り線) / ▦ (テーブル) / ⇄ (罫線変換) / 😊 (アイコン) / 📅 (日付) / 📋 (テンプレート) | カーソル位置に各記法を挿入 |

**挿入グループの特殊ボタン**（吹き出し型ポップアップを開く）:
- **▦ テーブルボタン** → `TablePicker`（8×8 グリッドで行×列を選択）
- **⇄ 罫線→Markdown 変換ボタン** → 選択範囲の罫線テーブル（`┌─┐│├┤┘└┴┬┼` や `+---+`）を Markdown テーブルに変換
- **😊 アイコンボタン** → `IconPicker`（カテゴリ別の絵文字一覧）
- **📅 日付ボタン** → 設定で指定したフォーマットで今日の日付を挿入
- **📋 テンプレートボタン** → `TemplatePicker`（`template/` フォルダのノート一覧から選択して挿入）

**ファイル**:
- `src/components/EditorToolbar.tsx` (本体)
- `src/components/TablePicker.tsx`
- `src/components/IconPicker.tsx`
- `src/components/TemplatePicker.tsx`
- `src/components/LinkPopover.tsx`

**CSS**: `.md-toolbar / .md-toolbar__group / .md-toolbar__divider / .md-toolbar__btn`

### 4.4 タグバー (`TagBar`)

EditorToolbar 直下、編集ビュー時のみ表示。ノートにメタデータとして紐づくタグをカンマ区切りで入力できる。

```
┌────────────────────────────────────────┐
│ 🏷 [#todo ×] [#work ×] [入力欄........] │
└────────────────────────────────────────┘
```

| 要素 | 名称 | 役割 |
|---|---|---|
| 🏷 アイコン | **タグバーアイコン** (tag-bar__icon) | 左端のラベル |
| バッジ | **タグバッジ** (tag-bar__badge) | 確定済みタグ。右の `×` で削除 |
| 入力欄 | **タグ入力欄** (tag-bar__input) | カンマで確定、Enter でも確定、空入力 BS で末尾削除 |

**確定ルール**:
- カンマ (`,` または `、`) を入力した瞬間、その手前の文字列が新しいタグになる
- Enter キーでも確定（IME 変換中の Enter は無視）
- 空入力でバックスペース → 末尾のタグを削除
- 重複タグは自動的に除外
- フォーカスアウト時にも未確定の draft をコミット

**保存**:
- ノートメタデータ (`notes.tags` カラム、JSON 配列) として SQLite に保存
- 保存先フォルダの `.md` 先頭の YAML front-matter にも `tags:` として書き込み（端末間で共有）
- `notes:list-tags` IPC で集計され、サイドバーのタグモードに表示される

**ファイル**: `src/components/TagBar.tsx`

### 4.5 編集 / プレビュー領域

| ビュー | コンポーネント | 内容 |
|---|---|---|
| 編集ビュー | `Editor` (CodeMirror 6) | マークダウンソース編集 |
| プレビュービュー | `Preview` (markdown-it) | レンダリング結果表示。先頭に **タグバッジ列**（`preview__tags`）があり、TagBar で設定したタグを `#tag` 形式で表示 |

**ドラッグ&ドロップ機能（編集ビューのみ）**:
- 画像 (PNG/JPG/...) → `images/<sha256>.<ext>` に保存 + 自動挿入
- PDF → `attachments/` 保存 + サムネイル付き挿入
- ZIP/LZH/7z → `attachments/` 保存 + 📎 リンクとして挿入

**プレビューでの操作**:
- 画像クリック → ライトボックスで拡大表示
- 添付リンククリック → OS の既定アプリで開く
- http(s) リンククリック → 既定ブラウザで開く
- **見出しアンカーリンク**: `[#見出し](#見出し)` をクリックでプレビュー内スクロール

**ファイル**:
- `src/components/Editor.tsx`
- `src/components/Preview.tsx`

## 5. AI チャットペイン (`AiChatPanel`)

メイン領域の右側に縦に並ぶ、LLM とのチャット用パネル。表示は任意（閉じる × ボタンで非表示）。

```
┌──────────────────┐
│ AI            ×  │ ← ai-chat__header
├──────────────────┤
│  user message    │ ← ai-chat__message--user
│ assistant 返答    │ ← ai-chat__message--assistant (Markdown 描画)
│  ## 見出し       │
│  - リスト        │
├──────────────────┤
│ [入力欄..........]│ ← ai-chat__input (textarea)
│ LLM: gpt-4o-mini  │
│              [送信]│
└──────────────────┘
```

| 要素 | 名称 | 役割 |
|---|---|---|
| ヘッダ | `ai-chat__header` | タイトル + 閉じるボタン |
| メッセージ領域 | `ai-chat__messages` | 履歴を縦並び表示 |

### 6.1 設定モーダル (`PreferencesModal`)

```
┌────────────────────────────────────────┐
│ 設定                                 × │
├────────────┬───────────────────────────┤
│ ▶ 基本     │  右ペインに詳細フォーム  │
│   コードブロック                       │
│   テンプレート                         │
│   セキュリティ                         │
│   保存先   │                          │
│   初期化   │                          │
└────────────┴───────────────────────────┘
```

| カテゴリ | 設定項目 |
|---|---|
| **基本** (general) | テーマ (ダーク/ライト) / メイン本文フォント (family / size) / サイドバーフォント / 日付フォーマット / 編集ボタンの表示 / 検索履歴の保存方式 / 検索履歴の件数 |

| **保存先** (storage) | 保存先フォルダ選択 / 既定に戻す / **データを上書き**（DB → disk 強制再書き出し） |
| **初期化** (reset) | テキストボックスに「初期化」入力 → 「初期化を実行」で DB クリア + 再起動 |

### 6.2 検索 / 置換ダイアログ

両方とも **非モーダル** + **ヘッダドラッグで自由移動** + **位置を記憶**:

```
┌─ 検索 ──────────── × ┐    ┌─ 置換 ──────────── × ┐
│ [検索文字列..........]│    │ [検索文字列..........]│
│                       │    │ [置換文字列..........]│
│ [前を検索][次を検索]   │    │ [次を検索][置換]      │
│             [閉じる]   │    │ [すべて置換][閉じる]  │
└─────────────────────┘    └─────────────────────┘
```

- Escape で閉じる
- 検索: Enter / Shift+Enter で次/前
- 置換: Enter で「次を検索」、置換時にカーソル前進

**ファイル**:

## 11. 指示を出すときの例

このリファレンスを使って、次のような指示が明確に伝わる:

- 「**編集ツールバーの挿入グループ** にカラーピッカーボタンを追加して」
- 「**サイドバーのファイルモード** で、フォルダの **ケバブメニュー** に "新しいサブフォルダ" を追加して」
- 「**ノートヘッダー** の右側、**ノートヘッダーのケバブボタン** にエクスポート項目を追加して」
- 「**設定モーダル** に新しいカテゴリ "外観" を追加して、その下にフォントサイズ設定を置いて」
- 「**サイドバーの検索モード** の **検索結果リスト** に、本文の抜粋（マッチ周辺）を表示して」
- 「**タブバー** の **タブ** に、未保存マークを表示して」
- 「**AI チャットペイン** の **アシスタントメッセージ** にコピーボタンを追加して」

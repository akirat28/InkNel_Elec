# InkNel — ノート同期機能仕様書

## 概要

InkNel のノート同期機能は、PC とスマートフォンなど複数のデバイス間で同一のノートの編集内容を共有するための仕組みです。

iCloud Drive / Dropbox / Google Drive のいずれか **1 つ** をクラウドストレージとして選択し、各サービスが提供する **ローカル同期フォルダ** を経由してファイルを共有します。OAuth やクラウド API は使用せず、OS の同期クライアントにネットワーク転送を委譲します。

---

## 対応クラウドストレージ

| プロバイダ | macOS での検出パス |
|---|---|
| iCloud Drive | `~/Library/Mobile Documents/com~apple~CloudDocs/` |
| Dropbox | `~/.dropbox/info.json` → `.path`、fallback: `~/Dropbox` 等 |
| Google Drive | `~/Library/CloudStorage/GoogleDrive-<email>/My Drive/` |

- **複数の同時利用は不可**。設定画面で 1 つだけ選択します。
- 「無効」を選ぶと同期機能は完全に停止します。

---

## クラウドフォルダの構成

選択されたクラウドストレージのルートに `InkNel/` ディレクトリが自動作成され、以下の構造で保存されます:

```
<クラウドフォルダ>/InkNel/
├── manifest.json            ← ノートのメタデータ一覧
├── notes/
│   ├── <uuid-1>.md          ← ノート本文
│   ├── <uuid-2>.md
│   └── ...
├── images/                  ← 画像ファイル (SHA-256 名)
│   ├── <sha256>.png
│   └── ...
└── attachments/             ← 添付ファイル (SHA-256 名)
    ├── <sha256>.pdf
    └── ...
```

### manifest.json

ノートのメタデータを JSON で管理します。同期時のタイムスタンプ比較に使用されます。

```json
{
  "version": 1,
  "lastSync": 1712851200000,
  "notes": {
    "<uuid>": {
      "title": "テスト",
      "folder": "work",
      "protected": false,
      "secret": false,
      "tags": ["tag1"],
      "createdAt": 1712800000000,
      "updatedAt": 1712850000000
    }
  }
}
```

---

## 同期の仕組み

### 基本原則

- PC 側のノートデータは **SQLite データベース** と **ローカルファイル** (`userData/notes/<id>.md`) の両方に保存されます。
- クラウド側のノートデータは **manifest.json** (メタデータ) と **notes/<id>.md** (本文) で構成されます。
- 同期の判定には **`updatedAt` タイムスタンプ** を使用します。ファイル内容の全体比較は行いません。

### タイムスタンプ比較のルール

| PC の `updatedAt` | クラウドの `updatedAt` | アクション |
|---|---|---|
| 同じ | 同じ | 何もしない（そのまま編集） |
| **PC が新しい** | 古い | PC → クラウドにコピー (push) |
| 古い | **クラウドが新しい** | クラウド → PC にコピー (pull) + DB 更新 |
| クラウドに無い | — | PC → クラウドにコピー (push) |
| — | PC に無い | クラウド → PC にコピー (pull) + DB 新規挿入 |

---

## 同期のタイミング

同期は以下の 3 つのタイミングで発生します。

### 1. ノート選択時（バックグラウンド個別同期）

**トリガー**: サイドバーでノートをクリックした瞬間

```
ユーザーがノートをクリック
  ↓
1. ローカル版を即座にロードして画面に表示
2. フッターに「確認中…」+ プログレスバー表示
3. バックグラウンドで checkAndSyncSingleNote() を呼び出し
  ↓
  ┌─ 同じ      → そのまま編集開始
  ├─ PC が新しい → PC のノートをクラウドにコピー → そのまま編集
  ├─ クラウドが新しい → クラウドからノートを取得
  │                   → DB に格納 (upsertNoteFromSync)
  │                   → 画面を最新内容で更新
  └─ 障害発生   → ブロック解除、ローカル版で操作を継続
```

- **UX**: ローカル版をまず表示するので遅延はほとんど感じません。
- **障害時**: ネットワークエラー等でクラウドにアクセスできない場合は、ブロックせずローカル版をそのまま使えます。

### 2. ノート編集時（ライトスルー即時反映）

**トリガー**: ノートの本文・メタデータ・フラグを変更した瞬間

```
ノート編集 → 300ms デバウンス → IPC ハンドラ
  ↓
1. writeBody(id, body)           ← ローカルファイル保存
2. touchNote(id)                 ← DB の updated_at 更新
3. pushSingleNote(provider, id)  ← クラウドフォルダに即時書き出し
  ↓
OS 同期クライアントがファイル変更を検知 → 自動でクラウドにアップロード
```

以下の操作すべてでライトスルーが動作します:

| IPC ハンドラ | 内容 |
|---|---|
| `notes:create` | 新規ノート作成 |
| `notes:update-body` | 本文変更 |
| `notes:update-meta` | タイトル・フォルダ・タグ変更 |
| `notes:set-protected` | 保護フラグ変更 |
| `notes:set-secret` | シークレットフラグ変更 |
| `notes:delete` | ノート削除（クラウドからも削除） |
| `images:save` | 画像追加 |
| `attachments:save` | 添付ファイル追加 |

### 3. 手動一括同期

**トリガー**: 以下の 2 パターン

- アクティビティバーの **共有ボタン** → サイドバー同期パネル → **「同期開始」ボタン**
- アプリ **起動時の自動同期**（共有が有効な場合）

```
全ノートを対象に双方向同期:
  1. manifest.json を読み込み
  2. ローカル DB から全ノート取得
  3. 全 ID (manifest ∪ DB) を反復:
     - ローカルのみ → push
     - クラウドのみ → pull
     - 両方あり → updatedAt 比較で push / pull / skip
  4. images/ ディレクトリの双方向コピー
  5. attachments/ ディレクトリの双方向コピー
  6. manifest.json を更新して書き出し
```

- 進捗はサイドバーのプログレスバーとフッターにリアルタイム表示されます。
- ノート処理ごとに event loop に制御を返すので、UI がフリーズしません。

## メディアファイル（画像・添付）の同期

画像 (`images/`) と添付ファイル (`attachments/`) は SHA-256 ハッシュをファイル名に使っています。

- **同名 = 同一内容**が保証されるため、タイムスタンプ比較は不要です。
- **ローカルにしか無い** → クラウドにコピー
- **クラウドにしか無い** → ローカルにコピー
- **両方にある** → スキップ（同一内容確定）

### ライトスルー

画像や添付を新たにドラッグ&ドロップした瞬間に、ローカル保存 + クラウドフォルダへのコピーが同時に行われます（`pushSingleMedia()`）。

## UI 上の同期状態表示

### アクティビティバー

- 共有が有効な場合、設定アイコンの上に **共有ボタン**（クラウド + 矢印アイコン）が表示されます。
- 同期中はアイコンが回転します。
- クリックするとサイドバーが「同期」モードに切り替わります。

### サイドバー同期パネル

```
┌──────────────────┐
│ [同期開始]        │ ← 手動同期ボタン
├──────────────────┤
│ プロバイダ  iCloud│
│ 状態      利用可能│ ← 現在のステータス
│ 最終同期  4/12 10:30│
│ クラウド件数   12件│
│ フォルダ  ~/...   │
├──────────────────┤
│ 進捗              │ ← 同期中のみ表示
│ ▓▓▓▓░░░░ 50%      │
│ ↑ テスト1 (3/8)   │
├──────────────────┤
│ 前回の結果        │ ← 完了後に表示
│ ノート ↑   3 件   │
│ ノート ↓   2 件   │
│ 変更なし   5 件   │
│ メディア ↑ 1 件   │
│ メディア ↓ 2 件   │
└──────────────────┘
```

### フッター（ステータスバー）

ウィンドウ最下部のフッターは 3 つの状態を表示します:

| 状態 | 左側の表示 | 右側の表示 |
|---|---|---|
| 通常時 | ノートのパス名 | ☁ 共有 |
| バックグラウンドチェック中 | ☁ 確認中… | 不確定プログレスバー |
| 一括同期中 | ↑/↓ ファイル名 (N/M) | 確定プログレスバー |

### ノート同期オーバーレイ

クラウドから pull して内容を更新中は、ノート領域全体に半透明の「同期中…」オーバーレイが表示され、操作がブロックされます。完了またはエラーで自動的に解除されます。

## 設定画面

設定 → 共有 から以下を設定します:

- **クラウド同期先**: 無効 / iCloud Drive / Dropbox / Google Drive
  - 各プロバイダの右側に「利用可能」「未検出」のステータスバッジ
  - 未検出のプロバイダはグレーアウトで選択不可
- **同期状態**: フォルダパス・最終同期日時・クラウド上のノート数
- **今すぐ同期ボタン**: 手動実行

設定は SQLite の settings テーブルに `share.provider` キーとして保存されます。

## 削除の同期

**現時点では部分的な対応です。**

| 操作 | 動作 |
|---|---|
| PC でノート削除 | ライトスルーによりクラウドの `.md` と manifest エントリも即座に削除 |
| クラウドで削除（他デバイスが削除） | 一括同期時に「クラウドに無い = ローカルのみ」と判定されて **再度 push** される（削除が伝播しない） |

将来的に tombstone（削除履歴）ベースの削除同期を導入することで、クラウド側の削除もローカルに反映できるようになります。

---

## 競合解決

**Last-write-wins 方式**を採用しています。

- 複数デバイスが同じノートを同時に編集した場合、`updatedAt` が最も大きい（= 最後に保存された）方が優先されます。
- マージは行いません。負けた側の内容は上書きされます。
- ノート単位の競合であり、ファイル内の行単位の競合検出は行いません。

---

## 障害発生時の挙動

| 障害 | 挙動 |
|---|---|
| クラウドフォルダが見つからない | 同期をスキップし、ローカル操作を継続 |
| manifest.json が破損 | 空の manifest として扱い、全ノートを push |
| ネットワーク切断（OS 同期クライアント停止） | ローカルのクラウドフォルダには書き込めるが、クラウドへのアップロードは OS 復旧後に自動再開 |
| ファイル読み書きエラー | 該当ノート/メディアをスキップし、他は続行 |
| バックグラウンドチェック失敗 | オーバーレイを解除し、ローカル版で操作を継続 |

**原則: 同期の障害でアプリ全体を止めない。ローカル操作は常に可能。**

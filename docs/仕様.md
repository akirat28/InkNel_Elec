# InkNel_Elec 仕様

## 1. アプリ概要

マークダウン形式のメモアプリ。PC（Electron）で編集し、スマートフォン（別途開発）で参照できることを最終目標とする。PCとスマートフォン間ではリアルタイムに同期する。

## 2. 機能要件

| # | 機能 | 概要 |
|---|---|---|
| F-1 | マークダウン編集 | CodeMirror 6 によるシンタックスハイライト付き入力 |
| F-2 | マークダウンプレビュー | markdown-it によるHTMLレンダリング |
| F-3 | タイトル単位のファイル分割 | 1メモ = 1 `.md` ファイル |
| F-4 | 全文検索 | SQLite FTS5 によるインデックス検索 |
| F-5 | 保存先選択 | ローカル / iCloud Drive / Google Drive / その他ネットワークボリューム |
| F-6 | スマートフォンとの同期 | リアルタイム同期（プロトコルは別途検討） |

## 3. 非機能要件

- macOS を最優先サポート、Windows / Linux はクロスプラットフォーム配慮
- 保存先は抽象化レイヤを設け、後から差し替え・追加ができる構造とする
- セキュリティ: Electronのcontext isolation有効、nodeIntegration無効、CSPを設定

## 4. 技術スタック

| 層 | 採用技術 |
|---|---|
| デスクトップランタイム | Electron |
| 言語 | TypeScript |
| UIフレームワーク | React |
| ビルド | Vite（electron-vite） |
| エディタ | CodeMirror 6（`@codemirror/lang-markdown`） |
| マークダウン変換 | markdown-it |
| データベース | SQLite（`better-sqlite3`） |

## 5. データ管理方針

- メモ本体は **マークダウンファイル（`.md`）** として保存先に保存（タイトル単位 = 1ファイル）
- メタデータ（タイトル、パス、更新日時、保存先種別、タグ等）と **検索インデックス（FTS5）** は **SQLite** に保持
- SQLite DB ファイルはユーザーデータディレクトリ（`app.getPath('userData')`）に配置
- DB操作はメインプロセス側でのみ実施し、レンダラからは IPC 経由で呼び出す

## 8. フェーズ計画

| フェーズ | 内容 |
|---|---|
| フェーズ1 | Electronウィンドウ起動 + マークダウン編集（CodeMirror 6）+ プレビュー（markdown-it）。タブ切替UI |
| フェーズ2 | SQLiteセットアップ + ローカルファイル保存 + ファイル一覧 + タイトル管理 |
| フェーズ3 | SQLite FTS5 による全文検索 |

## 8.1 フェーズ2 詳細設計

### 8.1.1 ストレージ配置

| 種別 | パス |
|---|---|
| メタデータDB | `${userData}/inknel.db`（SQLite） |
| ノート本文 | `${userData}/notes/${id}.md`（フラット配置） |
| 画像 | `${userData}/images/<sha256>.<ext>`（フラット配置、SHA-256命名で自動dedupe） |
| 添付ファイル | `${userData}/attachments/<sha256>.<ext>`（PDF/ZIP/LZH/7z等） |

`userData` は `app.getPath('userData')` 由来。macOS の場合は `~/Library/Application Support/InkNel/`。
画像と添付ファイルは notes と並列ディレクトリに配置することで、フェーズ4のストレージアダプタでまとめて同期可能。

### 8.1.2 SQLite スキーマ

```sql
CREATE TABLE IF NOT EXISTS notes (
  id         TEXT PRIMARY KEY,
  title      TEXT NOT NULL,
  folder     TEXT NOT NULL DEFAULT '',
  protected  INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_notes_folder  ON notes(folder);
CREATE INDEX IF NOT EXISTS idx_notes_updated ON notes(updated_at);

CREATE TABLE IF NOT EXISTS folders (
  path       TEXT PRIMARY KEY,
  created_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

- ノート本文は `.md` ファイル側にあり、DBには含めない
- フォルダ階層は `folder` 文字列（スラッシュ区切りの仮想パス）で管理
- **空フォルダ**（中にノートが1つもないフォルダ）は `folders` テーブルで管理する。これによりノートが無くてもフォルダ構造を維持できる
- ツリー描画時は `notes.folder` の集合と `folders.path` をマージし、両方のパスを階層化する
- FTS5 仮想テーブルはフェーズ3で導入

### 8.1.3 IPC チャネル

| チャネル | 入力 | 出力 | 用途 |
|---|---|---|---|
| `notes:list` | — | `NoteMeta[]` | 一覧取得（updated_at 降順） |
| `notes:create` | `{ title, folder?, body? }` | `NoteMeta` | 新規作成 |
| `notes:read-body` | `id` | `string` | 本文読み込み |
| `notes:update-meta` | `id, { title?, folder? }` | `NoteMeta` | タイトル/フォルダ更新 |
| `notes:update-body` | `id, body` | `void` | 本文保存 |
| `notes:set-protected` | `id, protected` | `NoteMeta` | 保護フラグの設定 |
| `notes:search` | `query` | `NoteMeta[]` | タイトルおよび本文の部分一致検索（大文字小文字無視） |
| `notes:delete` | `id` | `void` | 削除（DB行 + .md ファイル）。保護中のノートは Error |
| `images:save` | `ArrayBuffer, ext` | `string` | 画像を `images/<sha256>.<ext>` で保存（dedupe）、ファイル名を返す |
| `images:exists` | `filename` | `boolean` | 画像の存在確認 |
| `attachments:save` | `ArrayBuffer, ext` | `string` | 添付ファイルを `attachments/<sha256>.<ext>` で保存、ファイル名を返す |
| `attachments:open` | `filename` | `void` | OS の既定アプリで開く（`shell.openPath`） |
| `shell:open-external` | `url` | `void` | 外部 URL を既定ブラウザで開く（`shell.openExternal`） |
| `media:gc` | `{ images: string[], attachments: string[] }` | `{ deletedImages: string[], deletedAttachments: string[] }` | 候補のうち、どのノートからも参照されていないファイルを削除 |
| `folders:list` | — | `string[]` | 空フォルダ一覧（パス配列） |
| `folders:create` | `path` | `void` | 空フォルダを追加（既存ならno-op） |
| `folders:delete` | `path` | `void` | 空フォルダ定義を削除（ノートには影響しない） |
| `folders:rename` | `oldPath, newPath` | `void` | フォルダパスを書き換え。配下の全ノートと全サブフォルダを一括更新（トランザクション） |
| `settings:getAll` | — | `Record<string,string>` | 全設定をkey/value形式で取得 |
| `settings:set` | `key, value` | `void` | 設定値を保存（UPSERT） |

すべて `ipcMain.handle` / `ipcRenderer.invoke` ベース（Promise）で実装する。

### 8.1.4 レンダラ側の自動保存

- 編集中のノートは **本文・タイトル・フォルダ** ともに **300ms デバウンス** で自動保存する
- 別のノートに切り替える際は、保留中の保存を即座にフラッシュする

### 8.1.5 UI 追加

- **サイドバーヘッダにアイコンボタン2つ**:
  - 📄＋ **新規ノート**: 「無題」ノートを作成し選択状態に
  - 📁＋ **新規フォルダ**: パスを入力して空フォルダを作成（既存パスならツリー上で展開のみ）
- **編集領域上部に NoteHeader**：**スラッシュ区切りパス入力欄**（例: `階層1/テスト1`）、表示/編集セグメントトグル
  - 入力したパスはスラッシュの **最後の `/` を境に分割** され、前半が `folder`、後半が `title` として保存される
  - 例: `階層1/テスト1` → `folder="階層1"`, `title="テスト1"` → サイドバーに `階層1 > テスト1` の階層表示
  - 例: `階層1/階層2/テスト1` → 深いネストに対応
  - 例: `はじめに` → スラッシュ無しはルート直下の単独ノート
- **空状態**：1件もない場合は中央に「📄＋ でメモを作成しましょう」と表示

## 9. UI 仕様（フェーズ1）

### 9.1 全体レイアウト

画面を **3カラム** に分割する。最左のアクティビティバーで現在のビューを切り替え、
中央のサイドバーでファイル一覧を表示、右側に選択中ノートの編集/プレビュー/検索画面を表示する。

```
┌─────────────────────────────────────────────────────┐
│ [👁]│              │                                │
│ [✎]│  ファイル    │                                │
│ [🔍]│  一覧        │   メイン領域                    │
│    │  (Sidebar)   │   (Edit/Preview/Search)        │
│    │              │                                │
│ [⚙]│              │                                │
└────┴──────────────┴────────────────────────────────┘
 ↑
 ActivityBar (~48px)
```

- **ActivityBar (左端)**: VS Code風の縦アイコン列。上から「プレビュー」「編集」「検索」、最下部に「設定」
- **Sidebar (中央)**: ファイル一覧（ツリー）
- **Main (右側)**: 編集 / プレビュー / 検索 ビューの内容

### 9.1.1 ActivityBar の挙動

- 幅: 48px
- 上部アイコン（順番固定）:
  1. **ファイル** (📄): サイドバーを **ファイル一覧モード** に切り替える
  2. **検索** (🔍): サイドバーを **検索モード** に切り替える
- 下部アイコン:
  - **設定** (⚙): クリックで設定モーダルを開く（サイドバーモードは変えない）
- アクティブな上部アイコンには **左端に accent カラーのバー** で現在のサイドバーモードを示す
- 別のアイコンをクリックした場合はサイドバーの内容が切り替わるのみ。**メイン領域のノート表示はそのまま維持される**
- すでにアクティブなアイコンを再度クリックするとサイドバー自体が折りたたまれる（VS Code 同様）
- メイン領域は常にノートの編集/プレビューを表示する（編集ビューとプレビューの切替は NoteHeader 右端のトグルで行う）

### 9.2 サイドバー（ファイル一覧 / ツリー表示）

- メモを **ディレクトリ階層風のツリー** で表示する
- ツリーのフォルダノードはクリックで **展開 / 折りたたみ** が可能
- 葉ノード（ファイル）クリックで該当メモを右ペインに開く（フェーズ2以降で実装）
- フェーズ1ではプレースホルダ項目を表示する
- サイドバー幅の初期値: **240px**

#### 9.2.1 ディレクトリ階層の管理方針（重要）

実体ファイル自体はディレクトリに分けず、保存先の同一ディレクトリにフラットに配置する。
ディレクトリ階層は **各ファイルのメタ情報** として管理する。

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | ファイル一意ID |
| `title` | string | メモのタイトル（葉ノードの表示名） |
| `folder` | string | スラッシュ区切りのフォルダパス。例: `"work/projects"`。空文字列はルート |
| `updatedAt` | number | 更新日時（フェーズ2以降） |

- フォルダパスは **仮想的** なものであり、実ファイルシステム上のディレクトリとは対応しない
- フォルダの作成・移動・リネームは `folder` 文字列の編集で実現する
- メモ本体（`.md`）は保存先に **フラット** に配置され、メタ情報は SQLite に保持する（フェーズ2以降）
- フラット配置にすることでクラウド/ネットワーク同期や検索インデックス構築がシンプルになる

#### 9.2.1.1 サイドバーモード

サイドバーは ActivityBar のアイコン選択に応じて以下の2モードを切り替える:

| モード | ヘッダ | 内容 |
|---|---|---|
| `files` | 「ファイル」 | 仮想ディレクトリ階層のツリー（既定） |
| `search` | 「検索」 | 検索クエリ入力欄 + 「検索」ボタン + 結果リスト |

検索モードでは:
- 入力欄にクエリを入力し、検索ボタン押下または Enter キーで検索実行
- 結果は **タイトル一致 + 本文一致** の順で表示（重複は除外）
- 各結果はクリックでメイン領域に表示（保護中なら錠前アイコン付き）
- 検索結果のクリック時もファイルツリーと同様のセレクト挙動（保護フラグ尊重）

#### 検索履歴ナビゲーション

検索入力欄では矢印キーで過去の検索キーワードを呼び出せる:
- **`↑`**: 1つ古い履歴へ移動
- **`↓`**: 1つ新しい履歴へ移動（先頭に戻ると入力中だったドラフトを復元）
- 履歴は **重複を除去** し、新しい順に保持
- 保存件数の上限は設定の `search.historyLimit`（100 または 1000）
- 保存方式は設定の `search.historyMode`（`reset` / `persist`）

### 9.2.2 ツリー構築アルゴリズム

1. フラットなファイル配列を取得
2. 各ファイルの `folder` を `/` で分割しパスセグメント配列にする
3. ルートから順に階層をたどってフォルダノードを作成・取得
4. 末端フォルダの `files` 配列にファイルを追加
5. 同じ階層のフォルダ同士・ファイル同士はタイトル昇順でソート

#### 9.2.3 ツリーUI仕様

- フォルダノードは `▶`（折りたたみ）/ `▼`（展開）チェブロン + **フォルダアイコン** + フォルダ名
- ファイルノードは **ファイルアイコン** + タイトル
- 階層ごとにインデント（1階層 = 12px）
- フォルダ/ファイルアイコンは 14×14 の SVG で `currentColor` 描画。アクティブなファイルは accent カラーに追従
- ファイル行のインデントはチェブロン分 (`+16px`) を offset してファイルアイコンの x 座標がフォルダアイコンと揃うように調整
- フォルダの展開状態はUIステートとして保持（フェーズ2以降で永続化を検討）

#### 9.2.3.1 ドラッグ&ドロップでフォルダ移動

サイドバーのファイルツリーで、**ファイル行をフォルダノードへドラッグ&ドロップ** すると、そのファイルの `folder` フィールドがドロップ先フォルダのパスに更新される。

- **ドラッグ元**: 任意のファイル行 (`<button class="tree__file">`) を `draggable=true` にし、`dragstart` で `dataTransfer.setData('application/x-inknel-note-id', noteId)` を設定
- **ドロップ先**: 任意のフォルダ行 (`<button class="tree__folder">`) を以下のハンドラで対応
  - `dragover`: カスタム MIME タイプを検出して `preventDefault()` し、`dropEffect = 'move'`、視覚ハイライト
  - `dragleave`: ハイライト解除
  - `drop`: `getData('application/x-inknel-note-id')` で ID 取得、`onMoveNote(id, folderPath)` を呼ぶ
- **挙動**:
  - 同じフォルダへのドロップは no-op
  - 移動対象がアクティブノートの場合は編集中の `editingFolder` も同期更新
  - 保留中のメタ保存があれば事前に flush
  - 更新後にノート一覧を再取得してサイドバーが即座に再描画される
- **視覚フィードバック**: ドラッグ中のフォルダ行に `is-dragover` クラスが付与され、accent カラーの背景 + 点線アウトライン

- ファイル行にマウスホバーすると、行の **右端に縦三点リーダー（⋮）** のアイコンボタンが表示される
- 通常時は非表示、`:hover` 時のみ `opacity: 1` で表示
- ⋮ ボタンをクリックすると、ボタンの **右側に吹き出し型ポップアップメニュー** が開く
  - メニューの左端には三角ポインタ（吹き出しの突起）を CSS で描画
  - Reactのportalで `document.body` に直接レンダ
- メニュー項目（順番固定）:
  1. **✏️ 名称変更** — `RenameDialog` を開いて、ファイル名（パス形式）を編集
  2. **🔒 保護** / **🔓 保護解除** — 現在の保護状態に応じて表示。クリックで即時トグル
  3. **🗑 削除** — 確認ダイアログ後に削除。**保護中のノートではグレーアウトされ、クリック不可**
- 各項目の左にアイコン、右にラベルを配置
- メニューはメニュー外クリック / Escape で閉じる

#### 9.2.4.1 名称変更ダイアログ (`RenameDialog`)

- ケバブメニューから「名称変更」を選ぶと、`createPortal` で document.body にレンダされるモーダルダイアログが開く
- 入力欄には現在のファイル名（`buildPath(folder, title)` でパス形式に変換された値）が表示され、フォーカスと全選択がされる
- ユーザーが新しい名前を入力して **保存** を押すと、`parsePath` でパスを folder/title に分解し、`notes.updateMeta` で DB に書き込む
- アクティブノートを名称変更した場合は `editingTitle` / `editingFolder` も同期更新
- Enter キーで送信、Escape キーで閉じる
- 空名は保存ボタンが disabled になり、送信できない

#### 9.2.4.2 フォルダのケバブメニューと名称変更

- フォルダ行にもマウスホバーで右端に縦三点リーダー（⋮）が表示される
- メニュー項目は **「名称変更」のみ**
- クリックすると `RenameDialog` が開き、**フォルダの末尾セグメントだけ** が編集対象（例: `階層1/階層2` の `階層2` のみ）
- 保存時:
  - 親パス + 新しい末尾名 を結合して `newPath` を作る
  - `folders:rename(oldPath, newPath)` IPC を呼ぶ
  - メインプロセスで **配下の全ノート（`folder` が `oldPath` または `oldPath/...` のもの）** と **全サブフォルダ（`folders.path` が `oldPath` または `oldPath/...` のもの）** が一括更新される（SQLite トランザクション）
  - 例: `階層1` を `階層X` に変更 → `階層1`, `階層1/サブ`, `階層1/サブ/孫` 配下のすべてが `階層X`, `階層X/サブ`, `階層X/サブ/孫` に
- アクティブノートが影響を受けた場合は `editingFolder` も再同期

#### 9.2.5 ノートの保護

- 保護フラグは `notes.protected` カラム（`INTEGER NOT NULL DEFAULT 0`）で管理
- 保護中のノートは:
  - **サイドバーのファイル行の右端（ケバブボタンの左隣）に小さな錠前アイコンを常時表示**
  - コンテキストメニューの「削除」が無効化される
  - メインプロセス側でも `notes:delete` IPC が Error を返す（多重保護）
  - 編集ビューには直接遷移できず、PasswordDialog を経由する必要がある（9.6.5 参照）
- 編集自体は正しいパスワードで解錠すれば可能

### 9.3 サイドバーの折りたたみ

- TopBar の `[☰]` ボタンで サイドバーを **開閉** できる
- 折りたたみ状態はUI状態として保持（フェーズ2以降で永続化を検討）
- 折りたたみ時はサイドバーが画面から消え、右ペインが画面全幅に広がる
- アニメーション（width transition）で滑らかに開閉する

### 9.3.1 サイドバー幅のリサイズ

- サイドバー右端の **ドラッグハンドル** をマウスで左右にドラッグして幅を自由に変更できる
- 幅の制約: **最小 160px / 最大 480px**
- ドラッグ中はテキスト選択を無効化、カーソルを `col-resize` に変更
- ハンドル幅は約 4px、ホバー時にハイライト表示
- リサイズによる width 変更時はアニメーション（transition）を一時的に無効化し、引っかかりを防ぐ
- **リサイズ後の幅は SQLite の `ui.sidebarWidth` キーに保存**され、次回起動時に復元される
  - 変更時は 300ms デバウンスで保存
  - 復元時は min/max にクランプ
- サイドバー折りたたみ中はハンドルを非表示

### 9.4 編集／プレビュー領域

- メイン領域は選択中ノートの編集ビュー / プレビュービューを表示する
- 編集ビューは CodeMirror、プレビュービューは markdown-it
- ノートが1件もない場合は中央に空状態メッセージを表示

### 9.4.0 画像のドラッグ&ドロップ

エディタ画面に画像ファイルをドラッグ&ドロップして本文に貼り付けられる。

**動作**:
1. ユーザーがエディタに画像（PNG/JPG/JPEG/GIF/WebP/SVG/AVIF/BMP）をドロップ
2. 画像ファイルのバイナリを SHA-256 でハッシュ化、`images/<sha256>.<ext>` として `userData/images/` に保存（同一バイナリは自動的に dedupe）
3. ドロップ位置に `![元のファイル名](images/<sha256>.<ext>)` のマークダウンを挿入
4. プレビュー表示時、markdown-it の `renderer.rules.image` が `images/xxx` を `inknel-image://xxx` に書き換え
5. 画像はメインプロセスで登録された **`inknel-image://` カスタムプロトコル** 経由で読み込まれる

**設計上のポイント**:
- マークダウン本文には **相対パス** で記述するため、InkNel 外でも同一ディレクトリ構造があれば描画可能（移植性）
- カスタムプロトコルでファイル名を sanitize（`^[a-f0-9]{64}\.[a-z0-9]{2,5}$` のみ許可）し、path traversal を防ぐ
- IPC でのバイナリ転送は `ArrayBuffer` を使い、メイン側で `Buffer.from(data)` に変換
- 1 ファイルあたり **最大 25MB** までに制限

**未参照メディアの自動 GC（ガベージコレクション）**:

ノート編集中に画像や添付ファイル（PDF/ZIP/LZH/7z）を本文から削除した場合、対応する実ファイルも削除する。ただし以下の安全策を取る:

1. **トリガーは「編集→プレビューへの切替時」のみ**。編集中の Undo / 一時的な切り取りでは即座に削除しない
2. 切替時に `flushPendingSaves()` で保留中の保存を確定
3. アクティブノートを開いてから今までに本文に存在したメディア（`sessionImages` / `sessionAttachments`）の集合と、現在の本文に存在するメディアの集合を比較し、**消えたもの** を削除候補とする
4. 削除候補に対して **メインプロセスで全ノートを走査** し、他のノートから参照されていないことを確認してから削除（dedupe された同一ファイルが他のノートで使われている場合は残す）
5. GC 後、`sessionImages` / `sessionAttachments` を現在の状態にリセット

**仕様上の注意**:
- 編集中に画像を削除→プレビューに切替→編集に戻して Undo した場合、ファイルが既に削除されているため画像は復元されない（壊れた画像表示になる）。これはユーザーが認識して使うべき仕様
- 削除候補の正規表現は `images/[a-f0-9]{64}\.[a-z0-9]{2,5}` および `attachments/[a-f0-9]{64}\.[a-z0-9]{2,5}` のみマッチ。手書きの相対パスは GC 対象外

**プレビューでの表示サイズ**:
- **保存される実ファイル** はアップロード時のオリジナルサイズのまま（リサイズしない）
- プレビュー表示時のみ CSS で次の制約を適用:
  - **横幅 最大 1200px**
  - **縦幅 最大 500px**
  - **アスペクト比を保ったまま** スケール（`max-width` / `max-height` のみで指定し、`width: auto; height: auto;` のデフォルトを利用 → 両方の制約のうち厳しい方が自動適用される）
  - 表示サイズより小さい画像は **実寸表示**
- コンテナ幅が 1200px より狭い場合は `min(1200px, 100%)` でコンテナ内に収まるよう抑制
- マウスホバー時に `cursor: zoom-in` を表示し、クリック可能であることを示す

**画像ライトボックス（クリックで拡大）**:
- プレビュー領域の画像をクリックすると、画面全体に拡大表示される
- ライトボックス内では `max-width: 95vw / max-height: 95vh` でビューポートに収まる範囲で **オリジナルサイズに近い大きさ** で表示
- アスペクト比は保持
- 閉じる方法:
  - 背景（暗いオーバーレイ）をクリック
  - 右上の `×` ボタンをクリック
  - **Escape** キー
- 画像本体クリックは閉じない（誤操作防止）
- 開閉時にフェードアニメーション

### 9.4.0.1 添付ファイル（PDF / ZIP / LZH / 7z など）

画像以外のファイル（PDF/ZIP/LZH/LHA/7z）もエディタにドラッグ&ドロップで貼り付け可能。

**動作**:
1. ユーザーがエディタに対応拡張子のファイルをドロップ
2. SHA-256 でハッシュ化、`attachments/<sha256>.<ext>` として `userData/attachments/` に保存（dedupe）
3. ドロップ位置に `[元のファイル名](attachments/<sha256>.<ext>)` のリンクを挿入
4. プレビュー側で添付リンクは 📎 アイコン付きで表示
5. リンクをクリック → メインプロセスの `shell.openPath` で **OS の既定アプリ** で開く

**許可拡張子**: `pdf`, `zip`, `lzh`, `lha`, `7z`（将来追加可能）

**サイズ上限**: 1ファイルあたり **100MB**

**外部リンク（http/https）の挙動**:
- プレビュー内の `<a href="https://...">` クリックは `shell.openExternal` で **既定ブラウザ** に開く
- アプリ内のレンダラが他ページに遷移するのを防止

#### 9.4.0.2 PDF サムネイル

PDF をドロップした場合、**1ページ目のサムネイル PNG** を自動生成してリンクに貼り付ける:

1. レンダラ側で `pdfjs-dist` を使い、PDF バイナリの 1ページ目を canvas にレンダリング（最大幅 240px、アスペクト比保持）
2. canvas を PNG ArrayBuffer に変換
3. `images:save` IPC で `userData/images/<sha256>.png` に保存（dedupe）
4. マークダウンには **画像入りリンク**（ネスト記法）として挿入:
   ```markdown
   [![filename.pdf](images/<thumb-hash>.png)](attachments/<pdf-hash>.pdf)
   ```
5. プレビューでは `<a><img></a>` として描画され、サムネイルクリックで PDF が既定アプリで開く
6. サムネイル生成に失敗した場合（暗号化PDF など）は通常のリンク `[name](attachments/...)` にフォールバック

**実装上の注意**:
- pdfjs-dist の worker を Vite の `?url` インポートで読み込み、`GlobalWorkerOptions.workerSrc` に設定
- CSP の `worker-src` に `'self' blob:` を許可
- サムネイル生成は **drop 時の renderer プロセスで一度だけ**実行（保存後はマークダウンに記録されたパスからの読み出しのみ）
- 同一 PDF を再ドロップ → PDF と サムネイル両方が SHA-256 dedupe で再利用される

### 9.4.1 NoteHeader の表示/編集トグル

NoteHeader（タイトル行）の **右端に編集 / プレビューを切り替えるセグメントトグル** を配置する。
2 つのアイコンボタン（✎ 編集 / 👁 プレビュー）が並び、現在のビューに対応するボタンがアクティブハイライトされる。
インアクティブなボタンをクリックすると、そのビューに切り替わる。

```
┌────────────────────────────────────────────────────────┐

```

### 9.4.2 編集ツールバー

編集ビュー時かつ「編集ボタンの表示」設定がオンの時のみ、NoteHeader と Editor の間に **マークダウン挿入ツールバー** を表示する。
ツールバーには見出し/強調/リスト/挿入の各ボタンのみが並び、表示/編集トグルは含まれない（NoteHeader 側に統合済み）。

#### マークダウン挿入ボタン

| グループ | ボタン | 動作 |
|---|---|---|
| 見出し | `H1` `H2` `H3` `H4` `H5` `H6` | 行頭に `# ` 〜 `###### ` を追加 |
| 強調 | `B` (太字) | 選択範囲を `**…**` で囲む |
|  | `I` (斜体) | 選択範囲を `*…*` で囲む |
|  | `S` (打ち消し) | 選択範囲を `~~…~~` で囲む |
|  | `<>` (コード) | 選択範囲を `` `…` `` で囲む |
| リスト | `•` | 行頭に `- ` を追加 |
|  | `1.` | 行頭に `1. ` を追加 |
|  | `☐` | 行頭に `- [ ] ` を追加（タスクリスト） |
|  | `❝` | 行頭に `> ` を追加（引用） |
| 挿入 | `🔗` | `[テキスト](url)` を挿入 |
|  | `</>` | コードブロック ` ``` ` を挿入 |
|  | `―` | 区切り線 `---` を挿入 |
|  | `▦` (テーブルアイコン) | クリックで **テーブルサイズピッカー**（吹き出し型ポップアップ）を開き、最大8×8 のグリッドからマウスで行×列を選択して挿入 |
|  | `😊` (スマイリーアイコン) | クリックで **アイコンピッカー**（吹き出し型ポップアップ）を開き、カテゴリ別の絵文字一覧から選択して現在のカーソル位置に挿入 |

実装方針:
- `Editor` コンポーネントは `forwardRef` + `useImperativeHandle` で `insert` / `wrap` / `prefixLine` / `focus` を公開する
- ツールバーは Editor の ref を受け取り、各ボタンから対応するメソッドを呼ぶ
- `wrap()` は選択範囲がある場合は囲み、無い場合はプレースホルダを挿入して選択状態にする
- `prefixLine()` は複数行選択時にすべての行頭に prefix を追加する。単一行時は prefix 直後にカーソルを移動する

**アイコンピッカー（`IconPicker.tsx`）**:
- スマイリーアイコンクリックでボタンの真下に **吹き出し型のポップアップ** を `createPortal` で `document.body` に描画
- 上向き三角ポインタを CSS 擬似要素で配置
- 上部にカテゴリタブ（状態 / 文書 / 記号 / 顔 / 手 / 矢印 / 動物 / 食事 / 旅行）
- タブ下に 8 列のグリッドで該当カテゴリの絵文字一覧を表示
- 絵文字クリックで `insert(icon)` を呼んで現在のカーソル位置に挿入し、ピッカーを閉じる
- ピッカー外クリックまたは Escape で閉じる
- 絵文字は Unicode 文字列で挿入されるため、マークダウンとしてそのまま保存・プレビュー描画される

**テーブルピッカー（`TablePicker.tsx`）**:
- テーブルアイコンクリックでボタンの真下に **吹き出し型のポップアップ** を `createPortal` で `document.body` に描画
- 吹き出しには CSS 擬似要素 `::before` / `::after` で **上向きの三角ポインタ** を配置（ContextMenu と同じ二重三角の手法）
- 8×8 の小さなグリッドが表示され、マウスホバーで左上から現在位置までのセルが accent カラーでハイライト
- ラベルに「N 行 × M 列」を動的表示
- セルをクリックするとそのサイズの GFM テーブル雛形を `insert()` で挿入してピッカーを閉じる
- ピッカー外クリックまたは Escape で閉じる
- 挿入される雛形は ヘッダ行（`列1`, `列2`, …）+ 区切り行 + データ行 (rows-1) の形式

### 9.5 ウィンドウ

- 初期サイズ: 1200 × 800
- **二重起動を禁止する**: アプリは同時に1インスタンスしか起動できない。2つ目を起動しようとした場合は既存ウィンドウをフォーカスして終了する
- **ウィンドウサイズ・位置・最大化状態を SQLite に保存**し、次回起動時に復元する
  - キー: `window.bounds` (JSON `{x, y, width, height}`) と `window.maximized` (`'true' | 'false'`)
  - resize / move イベントを 300ms デバウンスで保存し、close 直前にも保存
  - 最大化中・最小化中・フルスクリーン中は bounds を上書きしない（最大化を解除した時の元のサイズを保つため）
  - 復元時、保存された bounds が現在のディスプレイ範囲外なら無視して既定位置で起動

#### 9.6.2 設定モーダル（2ペインレイアウト）

設定モーダルは **左ペインにカテゴリ一覧、右ペインに詳細フォーム** を持つ2ペイン構成。

| カテゴリ | 内容 |
|---|---|

#### 9.6.3 基本カテゴリの設定項目（順番固定、上から）

| キー | 型 | 既定値 | 説明 |
|---|---|---|---|
| `appearance.theme` | `'dark' \| 'light'` | `'dark'` | UI 全体の配色テーマ。ダーク（黒背景）/ ライト（白背景）を選択 |
| `editor.showInsertButtons` | bool | `true` | オン: 編集画面の上部に H1/H2/B/I 等のマークダウン挿入ボタンを表示。オフ: 編集ツールバー全体が非表示 |
| `search.historyMode` | `'reset' \| 'persist'` | `'reset'` | 検索履歴の保存方式。`reset`: アプリ再起動でリセット / `persist`: 再起動後も保持 |
| `search.historyLimit` | `100 \| 1000` | `100` | 検索履歴の最大保存件数。設定変更時に既存履歴も即時プルーン |
| `ui.sidebarWidth` | number | `240` | サイドバーの幅（px）。リサイズ時にデバウンス保存 |

検索履歴本体は SQLite の `settings` テーブルに `search.history` キーで JSON 文字列として保存される（`persist` モードの時のみ）。`reset` モード時はメモリ上にのみ保持される。

設定値は SQLite の `settings` テーブル（`key TEXT PRIMARY KEY, value TEXT`）に永続化する。
レンダラ側では bool は `'true'`/`'false'` 文字列で読み書きする。
アプリ起動時に `settings:getAll` で全設定を取得し、未設定キーは既定値を適用する。

#### 9.6.4 保護カテゴリの設定項目

| キー | 型 | 既定値 | 説明 |
|---|---|---|---|
| `protection.password` | string (4桁数字) | `'1234'` | 保護中ノートの編集を解錠するためのパスワード |

- パスワードは必ず4桁の数字（`/^\d{4}$/`）
- 設定モーダルの「保護」カテゴリで変更可能
- パスワード入力欄は `type="password"` でマスク表示（`••••`）
- アプリ初回起動時はDB未保存だが、`DEFAULT_SETTINGS` の `'1234'` がフォールバックとして適用される
- 現在の保存済みパスワードが既定値 `'1234'` のままの場合のみ、入力欄の下に「初期パスワード: 1234」のヒントを表示する。変更後は自動的に非表示になる

#### 9.6.5 ノート保護の動作

- ケバブメニューから「保護」を選ぶと、対象ノートの `protected` フラグが `true` になる
- 保護中ノートは:
  - **編集ビューに直接遷移できない**（NoteHeader の編集トグルクリックで PasswordDialog が開く）
  - 削除メニューはグレーアウト（既存仕様）
- **保護をかけた瞬間に編集ビューだった場合は、自動的にプレビュービューに切り替わる**
- パスワードダイアログ:
  - 4桁の数字入力欄 + 「解錠」ボタン
  - Enterキーでも送信可能
  - パスワードが正しければ:
    - 当該ノートを **セッション内で解錠** 状態にし、編集ビューに遷移
    - 解錠状態は別ファイルを選択するまで保持
  - パスワードが間違っていれば、ダイアログ内にエラーメッセージを表示し入力欄をクリア
- 保護解除（`保護解除` メニュー）でも `protected` フラグが false に戻り、編集制限がなくなる

#### 9.6.6 テーマ実装方針

- CSSカスタムプロパティ（`--bg`, `--fg`, `--accent` など）で配色を定義
- ダークがデフォルト (`:root`)、ライトは `[data-theme="light"]` セレクタで上書き
- アプリ起動時と設定変更時に `document.documentElement.dataset.theme` を更新
- CodeMirror 6 の `oneDark` テーマは `Compartment` で動的に reconfigure し、ライト時はテーマ未適用（CodeMirror デフォルトの白背景）にする
- Markdown プレビュー、サイドバー、アクティビティバー等は CSS 変数経由で自動的に切り替わる
